# ToyOS Memory Management Implementation Plan (4단계)

이 문서는 ToyOS의 물리 메모리 관리(PMM) 및 가상 메모리 관리(VMM) 구현 계획을 담고 있습니다.

## 1. 물리 메모리 관리자 (PMM)
UEFI 메모리 맵을 분석하여 실제 하드웨어 RAM의 빈 공간을 찾아 관리합니다.

### 관리 방식: 비트맵 (Bitmap)
- 메모리의 모든 4KB 페이지를 1비트로 나타냅니다. (1: 사용 중, 0: 비어 있음)
- 장점: 구현이 단순하고 메모리 단편화 상태를 파악하기 쉽습니다.
- 위치: 커널 바이너리 이후의 빈 메모리 공간에 비트맵을 생성합니다.

### 주요 기능
- `pmm_init()`: 메모리 맵을 스캔하여 비트맵 초기화.
- `pmm_alloc_page()`: 비어 있는 물리 페이지 하나를 할당.
- `pmm_free_page()`: 사용이 끝난 물리 페이지 해제.

## 2. 가상 메모리 관리자 (VMM)
64비트 4단계 페이징 시스템을 구축하여 메모리 보호 및 주소 변환을 수행합니다.

### 페이징 구조 (x86_64)
- **PML4** (Page Map Level 4)
- **PDPT** (Page Directory Pointer Table)
- **PD** (Page Directory)
- **PT** (Page Table)

### 매핑 전략: Identity Mapping (초기 단계)
- 초기에는 물리 주소와 가상 주소를 1:1로 일치시켜 복잡성을 줄입니다. (예: 가상 0x1000 == 물리 0x1000)
- 이후 커널 전용 고위 주소(Higher-half kernel) 매핑으로 확장할 수 있습니다.

## 3. 구현 단계
1. `src/kernel/pmm.c/h`: 비트맵 기반 물리 메모리 할당자 구현.
2. `src/kernel/vmm.c/h`: 페이징 구조체 정의 및 주소 매핑 로직 구현.
3. `kmain`: 메모리 매니저 초기화 및 할당 테스트.

## 4. 검증 방법
- 특정 물리 페이지를 할당받아 데이터를 쓰고 읽었을 때 값이 유지되는지 확인.
- 페이지 폴트(Page Fault)를 고의로 발생시켜 핸들러가 가상 메모리 문제를 감지하는지 테스트.
