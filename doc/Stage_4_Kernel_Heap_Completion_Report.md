# ToyOS 4단계 확장: 커널 힙(Kernel Heap) 구현 완료 보고서

## 1. 개요 (Overview)
기존의 페이지 단위(4KB) 메모리 관리 시스템을 확장하여, 바이트 단위로 메모리를 할당하고 해제할 수 있는 **커널 힙 할당자**를 성공적으로 구현했습니다. 이를 통해 커널은 보다 유연하게 동적 데이터를 관리할 수 있게 되었습니다.

## 2. 주요 구현 성과 (Key Achievements)

### 2.1 커널 힙 할당자 (`kmalloc`, `kfree`)
- **알고리즘**: 연결 리스트 기반의 First-fit 할당 방식을 채택했습니다.
- **기능**:
    - **Splitting**: 큰 블록을 할당할 때 필요한 만큼만 쪼개어 사용함으로써 메모리 낭비를 최소화했습니다.
    - **Coalescing**: 메모리 해제 시 인접한 가용 블록들을 하나로 병합하여 단편화(Fragmentation) 문제를 완화했습니다.
- **영역**: 가상 주소 `0x01000000` (16MB) 위치에 2MB 크기의 초기 힙 영역을 확보했습니다.

### 2.2 표준 타입 시스템 도입 (`stdint.h`, `stddef.h`)
- 기존에 직접 정의하여 사용하던 `uint64_t` 등의 타입을 컴파일러가 제공하는 Freestanding 헤더로 교체했습니다.
- 이를 통해 아키텍처별 타입 불일치 문제를 해결하고 코드의 표준 호환성을 높였습니다.

## 3. 해결된 기술적 문제 (Resolved Issues)
- **Type Conflict**: `kernel.h`와 표준 헤더 간의 `uint64_t` 중복 정의로 인한 컴파일 에러를 `<stdint.h>` 도입으로 해결했습니다.
- **VMM 연동**: 힙 초기화 과정에서 `PMM_AllocPage`와 `VMM_MapPage`를 연동하여 물리 메모리가 실시간으로 매핑되도록 처리했습니다.

## 4. 향후 계획 (Next Steps)
- **5단계: 인터럽트 및 키보드 드라이버**: 하드웨어 입력을 처리할 수 있는 환경을 구축합니다.
- **6단계: 멀티태스킹**: 프로세스 제어 블록(PCB)을 동적으로 할당(kmalloc 활용)하여 프로세스 관리를 시작합니다.
