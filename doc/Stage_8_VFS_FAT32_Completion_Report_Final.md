# VFS 및 FAT32 파일 시스템 구현 완료 보고서

ToyOS에 가상 파일 시스템(VFS) 계층을 도입하고 FAT32 파일 시스템 지원을 완료하였습니다.

## 변경 사항

### 1. 가상 파일 시스템 (VFS) 계층 (`vfs.h`, `vfs.c`)
- 파일 시스템의 종류에 관계없이 동일한 인터페이스(`read`, `readdir`)로 접근할 수 있는 `VFS_Node` 구조체를 정의하였습니다.
- 시스템 전역 루트 노드(`vfs_root`)를 관리하는 추상화 레이어를 구축하였습니다.

### 2. FAT32 파일 시스템 드라이버 (`fat32.h`, `fat32.c`)
- **BPB 파싱**: 디스크의 부트 섹터를 읽어 클러스터 크기, 데이터 영역 시작 위치 등을 계산합니다.
- **클러스터 체인 추적**: FAT 테이블을 참조하여 파일의 다음 데이터 클러스터를 찾아내는 로직을 구현하였습니다.
- **파일 및 디렉터리 읽기**: 8.3 포맷의 디렉터리 엔트리를 파싱하여 VFS 노드로 변환하고, 파일 데이터를 섹터 단위로 읽어오는 기능을 구현하였습니다.

### 3. 사용자 인터페이스 확장 (`shell.c`)
- **`ls` 명령어**: 현재 루트 디렉터리의 파일 및 디렉터리 목록을 출력합니다. (파일명, 타입, 크기 포함)
- **`cat <filename>` 명령어**: 지정한 파일의 내용을 읽어 화면에 출력합니다.

### 4. 커널 초기화 (`kernel.c`)
- IDE 드라이버 초기화 직후 `FAT32_Init()`을 호출하여 기본 파일 시스템을 마운트하도록 설정하였습니다.

## 검증 결과

### 빌드 및 실행
- WSL 환경에서 모든 소스 코드가 오류 없이 컴파일되었습니다.
- QEMU 실행 시 `FAT32: Mounted successfully` 메시지가 출력되었습니다.

### 기능 테스트
- **`ls` 실행 결과**: `iso` 폴더 내에 포함된 `kernel`, `BOOTX64.EFI` 등의 파일이 목록에 정상적으로 표시되었습니다.
- **`cat` 실행 결과**: 지정한 파일의 텍스트 데이터를 정상적으로 읽어 화면에 출력하는 것을 확인하였습니다.

## 향후 과제

- 파일 쓰기 기능 추가 (Write support)
