# GUI 시스템 구현 계획 (GUI System Implementation Plan)

## 1. 개요 (Overview)
ToyOS의 사용자 경험을 향상시키기 위해 텍스트 기반 콘솔(CLI)을 넘어선 그래픽 사용자 인터페이스(GUI) 시스템을 구축합니다. 본 계획은 화면 렌더링 최적화, 레이어 기반 윈도우 관리, 그리고 기본적인 UI 구성 요소를 포함합니다.

## 2. 주요 목표 (Goals)
- **더블 버퍼링(Double Buffering)** 적용으로 화면 깜빡임(Flickering) 제거.
- **레이어(Layer) 기반 관리**: 윈도우, 마우스 커서 등을 독립된 레이어로 취급하여 겹침 처리(Z-order) 지원.
- **마우스 상호작용**: 창 이동, 버튼 클릭 등 기본적인 마우스 이벤트 처리.
- **기본 UI 컴포넌트**: 창(Window), 버튼(Button), 레이블(Label) 등 핵심 요소 구현.

## 3. 상세 단계별 계획 (Phases)

### 1단계: 그래픽 엔진 고도화 및 더블 버퍼링
- `src/kernel/graphics.h` 및 `src/kernel/graphics.c` 생성.
- 기본 그리기 함수 구현:
    - `DrawPixel(x, y, color)`
    - `DrawRect(x, y, w, h, color)` (테두리 및 채우기)
- **더블 버퍼링 도입**:
    - VRAM과 동일한 크기의 시스템 메모리 버퍼(Back Buffer) 할당.
    - 모든 그리기 작업은 Back Buffer에서 수행.
    - `SwapBuffers()` 또는 `RefreshScreen()` 함수를 통해 VRAM으로 고속 복사.

### 2단계: 레이어 관리 시스템 (Layer Manager)
- **Layer 구조체** 정의:
    - 전용 버퍼 (그림 데이터)
    - 위치(x, y) 및 크기(w, h)
    - 투명도 처리 (Transparent Color Key)
    - 우선순위 (Z-order)
- **Layer Manager** 구현:
    - 여러 레이어를 관리하고 하단부터 순차적으로 Back Buffer에 렌더링.
    - 특정 레이어의 이동이나 내용 변경 시 해당 부분만 재렌더링하는 최적화 로직 고려.

### 3단계: 마우스 커서 및 이벤트 통합
- 기존 `mouse.c` 로직을 레이어 시스템으로 통합.
- 마우스 커서를 최상단 레이어로 할당하여 항상 다른 창 위에 표시되도록 처리.
- 마우스 좌표 이동에 따른 레이어 위치 업데이트 및 화면 갱신.

### 4단계: 윈도우 시스템 및 UI 구성 요소
- 기본적인 윈도우(창) 구조체 정의:
    - 타이틀 바, 닫기 버튼, 내용 영역(Content Area).
- 창 이동(Dragging) 기능 구현.
- 버튼 등 간단한 위젯(Widget) 클래스/구조체 및 클릭 이벤트 핸들러 구현.

## 4. 기대 효과
- 현대적인 운영체제의 느낌을 주는 시각적 인터페이스 제공.
- 다중 작업(Multitasking) 시각화 용이.
- 향후 그래픽 애플리케이션 개발을 위한 기반 마련.

## 5. 일정 및 우선순위
1. 더블 버퍼링 및 기본 그래픽 API (최우선)
2. 레이어 시스템 및 마우스 통합
3. 윈도우 관리자 프로토타입
