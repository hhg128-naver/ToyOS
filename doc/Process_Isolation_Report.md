# 유저 프로세스 격리 작업 완료 보고서 (User Process Isolation Completion Report)

ToyOS의 유저 모드 프로세스들이 서로 독립된 메모리 공간을 사용할 수 있도록 페이지 테이블 격리 작업을 완료하였습니다.

## 주요 변경 사항 (Changes Made)

### 가상 메모리 관리자 (VMM)
- **독립 주소 공간 생성**: `VMM_CreateAddressSpace()`를 통해 각 프로세스별로 새로운 **PML4**를 할당합니다.
- **매핑 유연성 확보**: `VMM_MapPageEx`를 구현하여 특정 PML4 테이블에 페이지를 매핑할 수 있게 되었습니다.
- **커널 공유**: 마스터 페이지 테이블의 엔트리들을 복사하여 모든 프로세스가 커널 코드와 하드웨어 자원(Framebuffer 등)을 공유하게 하였습니다.

### 태스크 시스템 (Task System)
- **PML4 추적**: `Task` 구조체에 `pml4` 필드를 추가하여 각 태스크가 자신의 주소 공간 주소를 보유합니다.
- **스케줄링 시 전환**: `Schedule()` 함수에서 태스크 전환 시 **CR3** 레지스터를 업데이트하여 주소 공간을 물리적으로 교체합니다.

## 테스트 및 검증 결과 (Validation)

### 빌드 결과
- WSL 환경에서 `make all` 명령을 통해 정상적으로 빌드됨을 확인하였습니다.

### 동작 검증 설계
- **동일 가상 주소 확인**: 두 개의 유저 태스크(Arg: 111, Arg: 222)가 동일한 가상 주소(`0x7FFFFFFF0000`)를 스택으로 사용하도록 설정하였습니다.
- **격리 확인**: 격리가 정상 작동한다면 두 태스크는 서로의 스택을 침범하지 않고 독립적으로 자신의 인자값을 출력하게 됩니다.

## 향후 작업 (Next Steps)
- **커널/유저 권한 분리**: 현재는 편의상 0~1GB 영역에 `PAGE_USER` 비트가 설정되어 있습니다. 보안을 위해 커널 전용 영역에서 이 비트를 제거하는 작업이 필요합니다.
- **동적 가상 메모리 할당**: 유저 공간의 힙 또는 데이터 영역을 동적으로 할당하는 메커니즘을 추가할 예정입니다.
