# ToyOS 향후 개발 계획 (ToyOS Future Development Plan)

## 1. 개요 (Overview)
기존 Legacy Multiboot 및 32비트 커널 아키텍처에서 64비트 UEFI 환경으로의 마이그레이션 및 핵심 커널 기능(메모리 관리, 인터럽트, 멀티태스킹) 구현이 완료되었습니다. 이제는 안정적인 시스템 서비스를 제공하고, 사용자 모드 전환 및 파일 시스템 지원을 통해 실제 활용 가능한 운영체제로 발전시켜야 합니다.

## 2. 개발 로드맵 및 진행 상황 (Roadmap & Status)

### 1단계: BootInfo 확장 및 UEFI 메모리 맵 핸드오버 [COMPLETED]
- **배경**: 커널이 물리 메모리 구성을 알아야 스스로 메모리 관리자를 초기화할 수 있음.
- **수행 항목**: `BootInfo` 구조체를 통해 Memory Map 및 프레임버퍼 정보를 커널로 전달.
- **결과**: 커널이 시스템의 전체 가용 메모리 영역을 완벽히 인지함.

### 2단계: 프레임버퍼 기반 텍스트 렌더링 (그래픽 콘솔) [COMPLETED]
- **배경**: UEFI GOP 환경에 맞는 픽셀 단위 문자 출력 시스템 필요.
- **수행 항목**: PSF 폰트 로더 및 텍스트 출력(`PrintString`, `Printf`) 구현.
- **결과**: 가독성 높은 폰트로 커널 메시지 출력 가능.

### 3단계: 프로세서 기초 환경 설정 (GDT/IDT) [COMPLETED]
- **배경**: 커널의 생명주기를 제어하기 위한 독자적인 Descriptor Table 구축.
- **수행 항목**: 64비트 Long Mode GDT/IDT 정의 및 Exception Handler 구현.
- **결과**: 시스템 예외 처리 및 하드웨어 인터럽트 수신 기반 마련.

### 4단계: 메모리 관리 시스템 (Memory Management) [COMPLETED]
- **배경**: OS 자체적인 메모리 자원 관리 및 가상 메모리 격리.
- **수행 항목**: PMM(Bitmap), VMM(4-Level Paging), Kernel Heap(kmalloc/kfree) 구현.
- **결과**: 동적 메모리 할당 및 프로세스 간 메모리 보호 기반 확보.

### 5단계: 인터럽트 및 키보드 입력 [COMPLETED]
- **배경**: 사용자와의 기초적인 상호작용 체계 마련.
- **수행 항목**: PIC/PIT 초기화 및 PS/2 키보드 인터럽트 핸들러 구현.
- **결과**: 키보드 입력을 통해 실시간 커널 반응 확인 가능.

### 6단계: 멀티태스킹 및 컨텍스트 스위칭 [COMPLETED]
- **배경**: 여러 작업을 동시에 실행할 수 있는 시스템 구축.
- **수행 항목**: Task 구조체 정의 및 타이머 기반 Preemptive Scheduling 구현.
- **결과**: 다중 태스크가 독립적인 스택을 가지고 병렬 실행됨.

## 3. 향후 중점 과제 (Next Milestones)

### 7단계: 유저 모드(User Mode) 전환 및 시스템 콜(System Call)
- **목표**: 커널 권한(Ring 0)과 사용자 권한(Ring 3)을 분리하여 안정성 강화.
- **수행 항목**: TSS(Task State Segment) 설정, 유저용 GDT 세그먼트 추가, `syscall/sysret` 또는 `int 0x80` 기반 인터페이스 구축.

### 8단계: 가상 파일 시스템(VFS) 및 FAT32 지원
- **목표**: 저장 장치로부터 데이터를 읽고 쓸 수 있는 영구 저장소 관리.
- **수행 항목**: 추상화된 VFS 계층 설계, UEFI 가상 디스크 또는 실제 디스크의 FAT32 파티션 파싱.

### 9단계: 고급 쉘(Shell) 및 유틸리티 구현
- **목표**: 사용자가 명령어를 입력하여 시스템을 제어할 수 있는 CLI 환경 완성.
- **수행 항목**: 명령어 파싱 엔진, 환경 변수, 기본 유틸리티(ls, cat, echo 등) 구현.

## 4. 검증 및 관리 (Verification & Maintenance)
- 각 단계 완료 시 `doc/` 폴더에 구현 보고서를 작성하여 기록 유지.
- `make run-uefi-debug`를 통한 지속적인 리그레션 테스트 수행.
