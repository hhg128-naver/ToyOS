# Stage 7: 유저 모드(User Mode) 전환 및 시스템 콜(System Call) 검증 계획서

## 1. 개요 (Overview)
본 단계의 목적은 이전 단계에서 구축된 GDT, TSS, MSR 등 하드웨어 인프라를 바탕으로, 실제 유저 권한(Ring 3)에서 코드를 실행하고 시스템 콜(System Call)을 통해 커널 서비스를 안전하게 호출하는 프로세스를 완벽히 구현하고 검증하는 것입니다.

## 2. 주요 목표 (Key Objectives)
1. **메모리 보호 (Paging)**: 유저 모드에서 접근 가능한 페이지에 `USER` 비트 권한 부여.
2. **유저 태스크 생성**: Ring 3 권한(Code/Data Segment)을 가진 태스크의 컨텍스트 초기화 로직 구현.
3. **유저 스택 관리**: 유저 모드 전용 스택 할당 및 TSS `rsp0` 연동을 통한 커널 스택 전환 보장.
4. **시스템 콜 핸들러 고도화**: `syscall` 명령어를 통해 전달된 요청을 처리하고 결과를 반환하는 C 언어 기반 핸들러 구현.
5. **실제 유저 모드 진입 및 검증**: 유저 모드 함수를 실행하여 시스템 콜을 호출하고 다시 유저 모드로 복귀하는 전체 사이클 확인.

## 3. 세부 구현 단계 (Implementation Steps)

### 3.1. 페이징 시스템(VMM) 업데이트
- **문제점**: 현재의 모든 페이지는 커널 전용(`Supervisor`)으로 설정되어 있어, 유저 모드에서 접근 시 Page Fault가 발생함.
- **수행 내용**: 
  - `vmm.h`에 `PAGE_USER (1 << 2)` 플래그 정의.
  - 유저 모드에서 사용할 메모리 영역(코드, 데이터, 스택, 힙) 매핑 시 `PAGE_USER` 비트를 추가하도록 관련 코드 수정.

### 3.2. 유저 모드 태스크 컨텍스트 초기화 (`task.c`)
- **수행 내용**: `CreateTask` 또는 새로운 `CreateUserTask` 함수를 구현하여 다음 사항을 설정.
  - **CS (Code Segment)**: 유저 코드 세그먼트 선택자 (`0x23`, RPL 3 포함).
  - **SS (Stack Segment)**: 유저 데이터 세그먼트 선택자 (`0x1B`, RPL 3 포함).
  - **RFLAGS**: `IF` (Interrupt Flag) 비트를 1로 설정하여 인터럽트 허용.
  - **RSP**: 별도로 할당된 유저 전용 스택 주소 설정.

### 3.3. TSS 및 커널 스택 전환 확인
- **수행 내용**: 유저 모드에서 인터럽트(타이머 등)나 예외 발생 시 CPU가 자동으로 커널 스택으로 전환할 수 있도록 설정 확인.
  - 컨텍스트 스위칭 시 현재 태스크의 `kernel_stack_top` 정보를 TSS의 `rsp0` 필드에 매핑하는 로직 점검.

### 3.4. 시스템 콜 핸들러 구현 (`syscall.c` 및 `asm_utils.asm`)
- **어셈블리 엔트리 (`SyscallEntry`)**: 유저 RSP를 저장하고 커널 RSP로 전환한 뒤, 레지스터들을 스택에 보존하고 C 핸들러로 점프.
- **C 핸들러 (`SyscallHandler`)**: 
  - `RAX` 레지스터의 시스템 콜 번호를 판별.
  - `RDI`, `RSI`, `RDX`, `R10`, `R8`, `R9` 순서로 전달된 인자를 추출하여 서비스 수행.
  - 처리 결과를 `RAX`에 담아 반환.

### 3.5. 유저 모드 테스트 코드 작성
- **수행 내용**: `kernel.c` 내부에 실제 Ring 3에서 실행될 `UserMain` 함수를 작성.
  ```c
  void UserMain() {
      // 인라인 어셈블리를 통한 syscall 호출 테스트
      const char* msg = "Hello from User Mode!\n";
      asm volatile (
          "mov $1, %%rax \n" // syscall 1 (write)
          "mov %0, %%rdi \n" // arg 1: string pointer
          "syscall"
          : : "r"(msg) : "rax", "rdi"
      );
      while(1); // 유저 모드에서 무한 루프
  }
  ```

## 4. 검증 시나리오 (Verification Scenario)
1. **부팅 후 스케줄러 기동**: 커널 초기화 완료 후 유저 태스크를 생성하여 런큐에 삽입.
2. **유저 모드 진입 확인**: `iretq` 또는 `sysretq`를 통해 유저 권한으로 성공적으로 전환되는지 확인 (QEMU 모니터링).
3. **시스템 콜 호출**: 유저 모드에서 실행 중인 코드가 `syscall`을 호출하여 커널 메시지를 출력하는지 확인.
4. **인터럽트 복귀 확인**: 유저 모드 실행 중 타이머 인터럽트가 발생했을 때, 커널 스케줄러가 정상 작동하고 다시 유저 모드로 돌아오는지 확인.

## 5. 향후 확장성 고려
- **ELF 로더**: 현재는 커널 내부에 유저 함수를 포함하지만, 향후 디스크에서 ELF 파일을 읽어 실행할 수 있는 구조로 확장.
- **VFS 연동**: 시스템 콜을 통해 파일 읽기/쓰기가 가능하도록 VFS 계층과 연결.
