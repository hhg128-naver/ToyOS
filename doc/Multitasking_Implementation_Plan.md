# 6단계: 멀티태스킹(Multitasking) 구현 계획

## 1. 개요 (Overview)
단일 실행 흐름에서 벗어나 여러 개의 태스크가 CPU를 나누어 사용하는 시분할 시스템을 구축합니다. 이를 위해 컨텍스트 스위칭과 기본적인 라운드 로빈 스케줄러를 도입합니다.

## 2. 설계 사양 (Design Specification)

### 2.1 프로세스 제어 블록 (PCB)
- **구조체 이름**: `Task`
- **필수 항목**:
    - `uint64_t rsp`: 태스크의 현재 스택 포인터 (컨텍스트 저장 위치)
    - `uint64_t id`: 태스크 식별자
    - `int state`: 태스크 상태 (READY, RUNNING, SLEEP 등)
    - `void* stack_base`: 할당된 스택 메모리의 시작 주소

### 2.2 컨텍스트 스위칭 (Context Switching)
- **방식**: 타이머 인터럽트 발생 시 현재 레지스터들을 스택에 푸시하고, `Task->rsp`를 업데이트한 뒤 다음 태스크의 스택에서 레지스터를 팝(Pop)하여 복원.
- **저장 대상**: 64비트 범용 레지스터 (RAX, RBX, ..., R15) 및 RIP, CS, RFLAGS 등.

### 2.3 스케줄러 (Scheduler)
- **알고리즘**: 라운드 로빈 (Round Robin)
- **관리 구조**: 태스크 포인터 배열 또는 연결 리스트.
- **트리거**: 타이머 인터럽트 (IRQ 0).

## 3. 구현 단계 (Phases)

### 1단계: 태스크 제어 구조 정의
- `src/kernel/task.h`, `task.c` 생성.
- `Task` 구조체 및 태스크 목록 관리를 위한 전역 변수 정의.

### 2단계: 컨텍스트 스위칭 유틸리티
- `src/kernel/asm_utils.asm`에 `SwitchContext` 추가.
- C 함수에서 호출 가능한 형태 또는 인터럽트 핸들러 내 직접 삽입 검토.

### 3단계: 태스크 생성 기능 (`CreateTask`)
- `kmalloc`을 사용하여 태스크별 커널 스택 할당.
- 스택의 초기 상태(시작 함수 주소, 레지스터 초기값)를 수동으로 설정.

### 4단계: 타이머 인터럽트 및 스케줄러 통합
- `pic.c`: 타이머 인터럽트(IRQ 0) 활성화.
- `interrupt.c`: IRQ 32(Timer) 발생 시 `Schedule()` 함수 호출.

### 5단계: 검증
- 서로 다른 메시지를 무한 루프에서 출력하는 두 개의 태스크가 번갈아 가며 실행되는지 확인.

## 4. 검증 시나리오
- [ ] 태스크 A와 태스크 B가 각각 독립적인 스택을 사용하는지 확인.
- [ ] 타이머 인터럽트 주기에 따라 화면의 출력이 교차로 발생하는지 확인.
- [ ] 태스크 전환 중 시스템 크래시(Triple Fault) 여부 점검.
